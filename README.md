# Revenue Funnel Build (Salesforce CRM Project)

## The Purpose of This Project

This project demonstrates CRM and Revenue Operations capability by building a complete end-to-end revenue funnel inside a Salesforce Developer Edition environment.

The goal was to simulate a real-world sales organization by:
- Importing enriched CRM datasets (leads, accounts, contacts, and opportunities)
- Configuring a structured revenue funnel across multiple pipeline stages
- Building dashboards and reports that translate raw CRM data into actionable revenue insights

The end result is a portfolio of dashboards that connect top-of-funnel activity to pipeline health and closed revenue, enabling clear visibility into conversion efficiency, pipeline risk, and revenue drivers.

---

## What is Contained in the Assets Folder

The `Assets` folder contains all project documentation and supporting visualization materials.

Each dashboard is organized in its own sub-structure:

- **2 Markdown files** describing a single dashboard per markdown file, including:
  - Dashboard purpose and business context
  - Key metrics and analytical breakdowns
  - Strategic insights derived from CRM data

- A **screenshots folder** containing images of all dashboard visualizations referenced in the Markdown file

This structure ensures each dashboard is fully self-contained, combining both narrative analysis and supporting visual evidence in one place.

## TLDR Version of Each Markdown File

### 1. Revenue Operations Command Center

This dashboard provides a high-level operational view of the entire revenue funnel, focusing on pipeline health, forecast reliability, channel performance, and team contribution.

#### Key Insights (TLDR)

- **Strong pipeline foundation but concentration risk exists**
  - A large portion of pipeline value is held by a small number of reps and channels, creating dependency risk but also highlighting clear best-practice performers.

- **Pipeline is healthy but not evenly distributed across stages**
  - Significant value is stuck in mid-funnel stages (especially Discovery), indicating the need for stronger deal progression and enablement support.

- **Forecast accuracy is limited by weak deal categorization**
  - A relatively small share of pipeline is confidently classified, reducing forecasting reliability and making revenue prediction less precise.

- **High-performing acquisition channels drive disproportionate revenue**
  - Referral and outbound channels (especially Cold Call) contribute high-quality pipeline and should be strategically reinforced.

- **Certain industries and segments underperform in revenue efficiency**
  - Some verticals generate lower average revenue per account, suggesting opportunities for reprioritization of sales focus.

---

### 2. Lead-to-Revenue Conversion Dashboard

This dashboard provides a full-funnel view from lead generation through opportunity conversion and closed revenue, focusing on how effectively leads convert into pipeline and revenue across industries and channels.

#### Key Insights (TLDR)

- **Lead response and qualification speed is a key bottleneck**
  - A large share of leads remain uncontacted or early-stage, creating avoidable stagnation in the top of the funnel.

- **Lead quality skews toward nurture-heavy segments**
  - Most leads are not immediately sales-ready, making structured nurturing and scoring essential for conversion efficiency.

- **Revenue is concentrated in a small number of high-performing industries**
  - A few verticals consistently generate the majority of revenue, indicating strong product-market fit in those segments.

- **Pipeline progression slows significantly in mid-funnel stages**
  - Deals tend to stall after early qualification, reducing overall conversion velocity and requiring stronger enablement between stages.

- **Channel performance is uneven but strategically important**
  - Some acquisition sources generate fewer deals but significantly higher deal sizes, meaning optimization should balance quality and volume rather than prioritize volume alone.
